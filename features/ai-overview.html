Daily SERP Scraping

Firecrawl + headless Chrome renders each tracked SERP daily. AI Overview block is detected via DOM selector heuristics plus structural pattern matching. Cited URLs are extracted and attributed to their owning domain.

Feature Extraction (22 features)

From cited and non-cited pages: entity density, schema markup presence (FAQPage, HowTo, Article), H-tag hierarchy, first-paragraph directness, definition density, reading level, content length, E-E-A-T signals (author byline, credentials, publication date), internal link depth, claim-source proximity, and image alt text coverage.

Logistic Regression Model

Trained on 50,000 URLs from our SERP snapshot corpus — labeled cited or not-cited for each AI Overview event. Regularization via L2 penalty (λ tuned via cross-validation). AUROC: 0.81 on holdout set. We chose logistic regression over gradient boosting deliberately: coefficients are interpretable, SHAP values are exact, and the model doesn't overfit niche SERPs the way tree models can.

SHAP Contributions Per Page

We compute Shapley values for each feature for your specific page. The output isn't "you should improve entity coverage" — it's "entity density is contributing −0.31 to your citation probability on this specific query; adding 4 named entity definitions would shift this feature from the 23rd to the 68th percentile, predicted citation probability shift: +0.09."

Honest Limits

What AI Overview Infiltrator doesn't do

The logistic regression model has AUROC 0.81 — meaning it's meaningfully predictive but not perfect. A page can implement all recommendations and still not get cited if Google's intent model changes or the AI Overview format for that query shifts. We can't force a citation — we can only improve the structural factors Google's model appears to weight. Model accuracy also varies by query category: health and legal SERPs have lower model confidence due to E-E-A-T complexity, and confidence is shown accordingly. We retrain the model monthly on new SERP corpus data.
