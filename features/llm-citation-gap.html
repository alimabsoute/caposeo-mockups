For Data Teams

Citation isn't ranking. It's a different signal stack — and we figured out which signals.

Step 1 — Citation Harvesting

We submit a parameterized query set to all four LLMs via API. Responses are parsed for citation patterns: explicit URL citations, domain mentions, paraphrased content attribution. Confidence on each attribution is scored — we don't count "probably from" as a citation.

Step 2 — Structural Feature Extraction

Cited and non-cited content is fetched and embedded via Voyage AI. We extract: entity density per 1K tokens, claim-source proximity (how close are cited facts to their source), H-tag hierarchy depth, schema markup types, definition density, first-paragraph answer completeness, reading level, and content depth proxies.

Step 3 — Structural Diff

We run a cosine similarity analysis between your content and the cited content in the same topic cluster. The gap report quantifies the delta on each feature: "cited content has 2.3× more named entities per 1K tokens than your content." The diff is factual, not qualitative.

Step 4 — Claude Sonnet Recommendation Pass

The structural diff is passed to Claude Sonnet with a structured output schema. It generates concrete, line-level edit recommendations ranked by SHAP-style feature contribution. "Add a definition block for [entity] in the second paragraph" — not "improve entity coverage."

Honest Limits

What LLM Citation Gap doesn't do

LLM citation patterns change with model updates — what Claude 3.7 cites may differ from Claude 4.x. We re-run baseline analysis after major model releases. Citation rate is a leading indicator of AI visibility, but it doesn't directly correlate to traffic until AI-cited links become more prominent in UI. This tool diagnoses the structural gap — it doesn't guarantee that implementing recommendations will raise citation rate within any specific timeframe. Results typically appear in 4–12 weeks post-implementation.
